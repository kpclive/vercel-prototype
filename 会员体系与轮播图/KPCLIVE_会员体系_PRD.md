---

## 一、项目背景与目标

KPCLIVE 围绕自有赛事 IP「KPC LIVE」开展德州扑克社区运营，提供比赛视频观看、直播、打卡、赛事活动、赛事资讯等服务。为进一步提升用户价值、构建商业化变现路径，规划上线会员体系。。

**本期目标**：
- 上线付费会员（VIP）核心能力，跑通付费 → 权益 → 续费的完整链路
- 支持运营侧通过「注册送会员」活动进行拉新转化
- 管理后台提供基础运营工具（手动赠送、延长会员）和数据看板
- 在架构上预留多等级、多来源（积分兑换等）的扩展能力

**非本期目标（明确不做）**：
- 在线棋牌游戏相关业务（公司红线）
- 自动续费 / 订阅制
- 多等级会员（仅上线 VIP，但保留扩展能力）
- 积分兑换会员（仅预留接口）

---

## 二、核心概念定义

| 概念 | 说明 |
|---|---|
| 会员等级 | 当前仅有 VIP 一级，数据结构需预留扩展（如 SVIP） |
| 会员有效期 | 用户当前的会员到期时间，所有渠道获取的时长一律累加到此字段 |
| 会员来源 | 标识每一笔会员时长的获取渠道，用于数据统计与追溯 |
| 普通用户 | 无会员或会员已到期的用户 |

**会员来源枚举（初版）**：
- `paid_purchase`：付费购买
- `register_gift`：注册赠送
- `manual_grant`：后台手动赠送
- `manual_extend`：后台手动延长
- `points_exchange`：积分兑换（预留，本期不上线）

## 三、功能需求详述

### 3.1 会员等级与权益

**3.1.1 等级**
- 当前仅上线 **VIP** 一个等级
- 数据库与后台需预留多等级扩展能力，未来新增 SVIP 等等级不需重构

**3.1.2 权益**
- 高清视频观看
- 直播观看
- 详细权益清单由独立文档维护，本 PRD 不展开

---

### 3.2 付费会员档位

**3.2.1 档位设置**

| 档位 | 时长 |
|---|---|
| 月度 | 1 个月 |
| 季度 | 3 个月 |
| 年度 | 12 个月 |

**3.2.2 价格配置**
- **计价货币：USD（美元）**
- 每个档位价格在管理后台可独立配置
- 价格调整即时生效，不需发版
- 用户端展示的价格实时同步后台最新配置
- 后台价格卡片仅展示档位名称、时长、价格输入框，不附加日均价、最近更新时间等冗余信息

**3.2.3 支付**
- 支持多种支付方式（具体由支付模块承接，本 PRD 不限定）
- 不支持自动续费 / 订阅，每次购买均为用户主动行为

**3.2.4 续费 / 叠加规则**
- 用户在会员有效期内再次购买，时长**直接累加**到现有到期时间之后
- 示例：剩余 10 天，购买月度（30 天），新到期时间 = 当前到期时间 + 30 天 = 40 天后

---

### 3.3 注册送会员活动

**3.3.1 配置项**（管理后台）

| 字段 | 类型 | 说明 |
|---|---|---|
| 活动总开关 | 开关 | 一键启用 / 关闭活动 |
| 活动开始时间 | 日期时间 | 活动生效起始时间 |
| 活动结束时间 | 日期时间 | 活动生效截止时间 |
| 赠送时长 | 数值（天） | 新注册用户获得的会员天数 |

**3.3.2 触发逻辑**
- 用户完成注册时，系统判断：
  - 活动总开关为「开」 **且** 当前时间在活动时间区间内 → 自动发放会员时长（来源标记为 `register_gift`）
  - 否则不发放
- 发放为自动到账，**用户无需手动领取**
- 活动结束后注册的新用户不再发放，**不支持补领**

**3.3.3 推广形式**
- 搭配 APP / PC 端轮播图进行广告推广（轮播图配置由运营位模块承接，本 PRD 不展开）

**3.3.4 用户已有会员时的处理**
- 注册赠送的时长在用户当前到期时间基础上累加（与 3.2.4 一致）

---

### 3.4 会员到期前提醒

**3.4.1 邮件提醒**
- 触发时机：会员剩余 7 天时
- 触发次数：**仅 1 次**
- 内容：提醒会员即将到期，引导续费
- 邮件模板由消息中心维护

**3.4.2 APP 内弹窗提醒**
- 触发时机：会员剩余 7 天时，用户启动 APP 时
- 触发次数：**仅 1 次**（用户关闭后不再弹出）
- 内容：提示到期时间，引导跳转续费页

**3.4.3 到期处理**
- 会员到期时间到达后，用户身份直接变为「普通用户」
- **不保留「曾是会员」标识**
- 不发送到期通知

---

### 3.5 用户端展示

**3.5.1 APP 端**

会员状态展示位（个人中心或会员中心页）：
- VIP 标识（图标 / 角标）
- 会员到期时间
- 「开通 / 续费」入口按钮
  - 未开通用户显示「开通会员」
  - 已开通用户显示「续费」

**3.5.2 PC 官网**

与 APP 端展示信息一致：
- VIP 标识
- 到期时间
- 开通 / 续费入口

**3.5.3 普通用户**
- 不显示 VIP 标识
- 显示「开通会员」入口

---

### 3.6 管理后台 - 会员管理

#### 3.6.1 价格配置
- 月度 / 季度 / 年度三档价格独立可配置，**计价货币统一为 USD**
- 保存即生效
- 价格卡片展示精简：仅显示档位、时长、价格输入框，不展示日均价、最近更新时间等附加信息

#### 3.6.2 注册送会员活动配置
- 详见 3.3.1
- 配置项保存即生效

#### 3.6.3 手动赠送会员

**适用场景**：仅对 **非会员用户**（无会员或会员已过期）使用，用于首次开通会员或为已过期用户重新开通。

**操作功能**：
- 选择目标用户（支持按用户 ID / 手机号 / 用户名等查询）
- 输入赠送时长（天数）
- 填写备注（可选，便于后续追溯）
- 提交后立即生效，从当前时间起记为新的会员开通时间

**业务规则**：
- 若目标用户**当前已是会员**（VIP 未到期），系统**拒绝操作并提示失败**："该用户已是会员，请使用『手动延长』功能"
- 提示形式：表单内联错误提示 + Toast 弹出
- 拦截发生在前端校验和后端二次校验两层，防止绕过

**权限**：
- 需具备会员管理权限的运营人员
- **无需审批流程**

**历史记录**：
| 字段 | 说明 |
|---|---|
| 操作时间 | 自动记录 |
| 操作人 | 后台账号 |
| 目标用户 | 用户 ID / 用户名 |
| 赠送时长 | 天数 |
| 备注 | 操作人填写 |
| 操作前到期时间 | 系统自动记录（多为"无会员"） |
| 操作后到期时间 | 系统自动记录 |

#### 3.6.4 手动延长会员时间

**适用场景**：仅对**当前 VIP 用户**（会员未到期）使用，用于在现有到期时间基础上追加时长。

**操作功能**：
- 选择目标用户（需当前为 VIP 状态）
- 输入延长时长（天数）
- 填写备注（可选）
- 提交后立即生效，时长累加到当前到期时间之后

**业务规则**：
- 若目标用户**当前非会员**（无会员或已过期），系统**拒绝操作并提示失败**："该用户非会员，请使用『手动赠送』功能"
- 提示形式同手动赠送：表单内联错误提示 + Toast

**职责划分说明**：
| 用户当前状态 | 应使用的功能 |
|---|---|
| 无会员 / 会员已过期 | 手动赠送 |
| 当前 VIP（会员有效期内） | 手动延长 |

底层数据均为"累加到期时间"，但通过功能入口区分使用场景，便于运营操作直觉和数据统计区分（来源标记为 `manual_grant` / `manual_extend`）。

**权限与历史记录**：同 3.6.3

#### 3.6.5 用户管理（会员列表）

**功能定位**：会员用户的统一查询入口，支持快速查看任意用户的会员状态并发起赠送 / 延长操作。

**列表字段**：
- 用户信息（用户名、UID、手机号）
- 等级（VIP）
- 开通时间
- 到期时间
- 剩余天数
- 主要来源（付费购买、注册赠送、手动赠送等）
- 状态（生效中、即将到期、已过期）
- **操作**（赠送 / 延长 按钮）

**筛选条件**：搜索框（用户名 / 手机号 / UID）、等级、来源、状态。

**快捷操作交互规则**：
| 用户当前状态 | 「赠送」按钮 | 「延长」按钮 |
|---|---|---|
| 当前非会员 | 可点击，跳转到手动赠送页并预填用户 | 禁用，点击提示"该用户非会员，请使用赠送" |
| 当前 VIP | 禁用，点击提示"该用户已是会员，请使用延长" | 可点击，跳转到手动延长页并预填用户 |

**跳转逻辑**：
- 点击赠送 / 延长按钮后，自动跳转到对应功能页
- 目标用户信息自动预填到"已选用户"卡片
- 运营无需再手动搜索一次目标用户

#### 3.6.6 历史记录查询
- 手动赠送 / 延长操作均有完整历史记录
- 后台可查询、按条件筛选（用户、操作人、时间范围）
- **本期不支持导出**

---

### 3.7 管理后台 - 数据看板

简洁实用为原则，提供以下三类核心指标：

**3.7.1 会员规模**
| 指标 | 说明 |

| 新增会员数 | 支持按日 / 周 / 月切换 |
| 到期会员数 | 支持按日 / 周 / 月切换 |

**3.7.2 营收**
| 指标 | 说明 |

| 按档位营收分布 | 月度 / 季度 / 年度各档位营收占比 |

**3.7.3 会员来源**
| 指标 | 说明 |
|---|---|
| 各来源占比 | 付费、注册赠送、手动赠送 |
| 积分兑换 | 占位字段，本期数据为 0 |

## 五、关键业务规则汇总

为便于开发与测试快速对齐，关键规则集中列出：

1. **时长累加规则**：所有渠道（付费、注册赠送、手动赠送、手动延长、未来的积分兑换）获取的会员时长，一律累加到用户当前到期时间之后。
2. **到期处理**：会员到期立即变为普通用户，无过渡状态，无保留标识。
3. **提醒规则**：剩余 7 天时邮件 + 弹窗各提醒 1 次，不重复触发。
4. **注册赠送**：仅在活动开关开启 + 时间区间内的新注册用户自动获得，不可补领。
5. **后台操作**：手动赠送 / 延长无需审批，但必须留痕。
6. **价格**：后台配置实时生效，无需发版。
7. **赠送 / 延长职责划分**：
   - 手动赠送 → 仅适用于**非会员**用户（无会员或已过期）
   - 手动延长 → 仅适用于**当前 VIP** 用户
   - 越界操作系统拒绝并提示失败，引导切换到对应功能。

# 第二部分：轮播图模块

## 八、轮播图模块概述

### 8.1 模块定位

轮播图（Banner）是 KPCLIVE APP 与 PC 官网首页顶部的内容广告位，承载赛事推广、活动宣传、会员转化等运营目标。本模块在管理后台「内容运营」分组下，提供轮播图的增删改查、上下线、定时发布、受众定向等能力。

### 8.2 本期范围

**做**：
- 1 个轮播位（首页轮播图）
- APP 端 / PC 端各自独立管理
- 跳转、受众、排序、定时上下线
- 数量不限制

**不做（本期）**：
- 多轮播位（社区页、详情页等）
- 曝光 / 点击数据统计
- A/B 测试
- 审批流程

---

## 九、功能需求详述

### 9.1 端区分

| 端 | 独立性 | 建议尺寸 |
|---|---|---|
| APP 端 | 完全独立管理一套轮播图 | 750 × 360 |
| PC 端 | 完全独立管理一套轮播图 | 1920 × 600 |

- 后台列表通过端切换 Tab（APP 端 / PC 端）查看与操作
- 同一张图若需在两端展示，需分别创建并上传对应尺寸的素材
- 两端的轮播图独立计数、独立排序、独立上下线

### 9.2 单张轮播图字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| ID | 系统生成 | — | 唯一标识（如 B001） |
| 展示端 | 单选 | 是 | APP / PC |
| 图片 | 图片上传 | 是 | JPG / PNG，APP 端 ≤ 2MB，PC 端 ≤ 5MB |
| 标题 | 文本 | 是 | 仅后台内部标识，不展示给用户 |
| 跳转类型 | 单选 | 是 | 无跳转 / APP 内部页面 / 外部 H5 链接 / 资讯详情 |
| 跳转目标 | 联动配置 | 视类型 | 详见 9.3 |
| 受众 | 单选 | 是 | 全部用户 / 仅会员 / 仅非会员 |
| 排序权重 | 数字 | 是 | 数字越大越靠前，默认 100 |
| 开始时间 | 日期时间 | 否 | 留空表示立即生效 |
| 结束时间 | 日期时间 | 否 | 留空表示永久生效 |
| 状态 | 枚举 | 系统生成 | 详见 9.6 |
| 创建人 / 创建时间 | 系统记录 | — | 用于追溯 |

### 9.3 跳转配置

跳转类型采用**二级联动**配置：第一级选类型，第二级根据类型出现对应的目标选择控件。

| 第一级（跳转类型） | 第二级（目标选择） | 说明 |
|---|---|---|
| 无跳转 | — | 纯展示，点击无响应 |
| APP 内部页面 | 先选页面类型，再选具体内容 | 详见 9.3.1 |
| 外部 H5 链接 | 填入完整 URL | 需以 http:// 或 https:// 开头 |
| 资讯详情 | 从已发布资讯中搜索选择 | 引用资讯模块数据 |

#### 9.3.1 APP 内部页面（枚举值）

| 页面类型 | 第二级选择内容 |
|---|---|
| 赛事详情 | 选择具体赛事（搜索 / 列表） |
| 视频详情 | 选择具体视频（搜索 / 列表） |
| 会员开通页 | 固定页面，无需选择 |
| 活动页 | 选择具体活动 ID |

> 跳转目标的具体内容选择依赖赛事、视频、资讯等业务模块的列表接口，需与对应模块对齐数据返回结构。

### 9.4 受众定向

每张轮播图必须选择一个受众范围：

| 受众 | 展示对象 | 典型场景 |
|---|---|---|
| 全部用户 | 所有访问首页的用户（包括未登录） | 通用宣传、赛事推广 |
| 仅会员 | 当前 VIP 状态有效的用户 | 会员专属活动、VIP 直播 |
| 仅非会员 | 未开通会员或会员已过期的用户 | 注册送会员推广、付费转化 |

**判定时机**：用户首页加载时根据当前会员状态实时判定，不缓存。

**互不可见**：受众外的用户完全看不到该轮播图，不占位、不显示。

### 9.5 排序

- 排序方式：**数字权重**
- 取值：非负整数，默认 100，建议步进 10 便于插入
- 规则：**权重数字越大越靠前**；权重相同时按创建时间倒序
- 后台操作：列表行内的权重输入框可直接修改，失焦后实时生效

### 9.6 上下线与生效逻辑

#### 9.6.1 状态定义

| 状态 | 触发条件 |
|---|---|
| 未开始（scheduled） | 已配置开始时间，但尚未到达 |
| 已上线（online） | 当前在生效时间区间内，且手动状态为"上线" |
| 已下线（offline） | 在生效时间区间内但被手动下线 |
| 已结束（ended） | 已到达结束时间 |

#### 9.6.2 自动化规则

- 到达开始时间：状态自动从「未开始」→「已上线」
- 到达结束时间：状态自动 →「已结束」（不消失，保留在列表中可查）
- 未填开始时间：保存即上线
- 未填结束时间：永久生效，直到手动下线

#### 9.6.3 手动操作

- 「上线 / 下线」按钮：在「已上线」与「已下线」之间切换
- 「已结束」状态不可手动上线（需先编辑结束时间）
- 「未开始」状态不可手动上线（需先编辑开始时间或留空）

### 9.7 列表与筛选

**列表展示字段**：缩略图、标题、跳转、受众、权重（可编辑）、生效时间、状态、操作

**筛选项**：状态、受众

**行内操作**：
- 编辑：打开编辑抽屉
- 上线 / 下线：见 9.6.3
- 删除：二次确认后永久删除

### 9.8 新建 / 编辑交互

采用**右侧抽屉**形式，包含字段：展示端 → 图片 → 标题 → 跳转类型 → 跳转目标（联动）→ 受众 → 排序权重 → 开始时间 → 结束时间。

**保存校验**：
- 图片、标题、跳转类型、受众为必填
- 跳转类型非"无跳转"时，跳转目标必填
- 外部 H5 链接需校验 URL 格式
- 开始时间晚于结束时间时拒绝保存

### 9.9 与「注册送会员」活动的协同

注册送会员活动期间，运营通常配置一张「仅非会员」受众的轮播图配合推广，两个模块之间无强绑定关系，由运营手动协同。建议在 PRD 后续迭代中考虑：

- 创建活动时同步引导创建配套轮播图（TBD）
- 活动结束时同步下线关联轮播图（TBD）

## 十、关键业务规则汇总（轮播图）

1. **端隔离**：APP / PC 完全独立，互不影响
2. **数量**：不设上限，运营自行控制
3. **受众判定**：按用户实时会员状态判定，受众外用户不可见
4. **排序**：权重越大越靠前，相同权重按创建时间倒序
5. **生效时间**：开始 / 结束均非必填；不填则立即生效或永久生效；到达结束时间自动归档为"已结束"，不删除
6. **删除**：删除即永久消失，无回收站
